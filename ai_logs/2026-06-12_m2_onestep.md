# 2026-06-12 - M2 naive one-step baseline

## Goal
Show that the frozen M1 EMA teacher, run as a one-step generator (single x0
estimate from Gaussian noise), is much worse than multi-step sampling. This is a
diagnostic baseline that motivates DMD. No training, no DMD/DMD2.

## Tools
- ChatGPT: refined the M2 plan and Codex prompt (see [chatgpt_planning.md](chatgpt_planning.md)).
- Codex: implemented the one-step primitive, fixed eval noise, the M2 entry point, figures, and docs.

## Files touched
- New: `src/baseline_onestep.py` (M2 entry point), `configs/baseline.yaml`.
- Edited: `src/diffusion.py` (`onestep_x0` primitive; `p_sample_loop` accepts a fixed `x_init`), `src/utils.py` (`get_eval_noise`), `src/plotting.py` (`plot_compare_triptych`, `plot_mode_counts`), `run.sh` (`m2` case), `README.md`, `ai_logs/index.md`.

## Design choices
- `onestep_x0` is the M2 naive one-step teacher baseline (one teacher eps -> x0_hat via DDPM Eq. 15 at `t = T-1`). It is NOT the DMD/DMD2 student; M3/M4 train separate students.
- Load the EMA teacher only (`load_teacher(which="ema")`), `.eval()`, all sampling/metrics under `torch.no_grad()`.
- Reproducibility: one-step and multi-step share the same fixed `x_T`, saved on CPU at `outputs/eval_noise.pt` and reused by M3/M4; the stochastic reverse sampler is seeded with `torch.manual_seed(seed)` before `p_sample_loop`.
- M1 teacher and its outputs are untouched.

## What was checked
- `./run.sh m2` runs clean from the existing checkout and writes the four `outputs/m2/` artifacts plus `outputs/eval_noise.pt`.
- Metrics (vs true `pi_1`, n=2000, `T=200`):
  - multi-step EMA: 8 modes, entropy 0.999, off-support 0.124, ED 0.0053, MMD 0.0014
  - one-step EMA: 7 modes, entropy 0.558, off-support 0.443, ED 0.311, MMD 0.244
  - degradation (one - multi): off-support +0.318, ED +0.305, MMD +0.242
- `one_step_clearly_worse = True`.

## Honest takeaway
The one-step baseline clearly fails: it drops a mode and collapses entropy and all
distances. A single jump from noise to data cannot represent the multimodal
target, which is exactly the gap M3 (DMD) must close.
