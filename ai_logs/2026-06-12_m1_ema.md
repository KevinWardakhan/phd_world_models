# 2026-06-12 - M1 EMA teacher

## Goal
Add EMA (exponential moving average) of the DDPM denoiser weights to improve
teacher sampling stability/quality, and use the EMA model as the reported frozen
teacher. No M2/DMD/DMD2.

## Tools
- ChatGPT: refined the EMA plan and Codex prompt (see [chatgpt_planning.md](chatgpt_planning.md)).
- Codex: implemented the EMA helper, training integration, dual evaluation, checkpoint, and docs.

## Files touched
- New: `src/ema.py` (small `EMA` class: deepcopy, no-grad update, state_dict).
- Edited: `src/train_teacher.py` (update EMA after each step; evaluate raw and EMA; EMA is the headline teacher for figures/GIF/snapshots/checkpoint), `src/diffusion.py` (`load_teacher(which=...)` restores EMA by default), `configs/teacher.yaml` (`ema` section), `README.md`, `ai_logs/index.md`.

## Design choices
- Standard EMA, `decay=0.999`, `start_step=0`, config-driven. EMA does not touch the optimizer step.
- The EMA model is the frozen teacher M2-M4 load (`load_teacher(which="eval")`); raw weights kept in the checkpoint for debugging/reproducibility.
- Checkpoint stores `model_state_dict`, `ema_state_dict`, `model_config`, `diffusion_config`, `data`, `config`, `metrics`, `ema_used_for_eval`.

## What was checked
- `teacher.pt` contains both raw and EMA state dicts; `ema_used_for_eval=True`; `load_teacher(which="eval")` returns EMA weights.
- Raw vs EMA (vs true `pi_1`, n=2000, `T=200`):
  - entropy 0.9987 -> 0.9988 (marginal)
  - off-support 0.144 -> 0.142 (marginal)
  - energy distance 0.00207 -> 0.00148 (-28%)
  - MMD 0.00105 -> 0.00083 (-21%)
- All acceptance checks still pass; headline teacher = EMA.

## Honest takeaway
EMA clearly improves the distributional distances (ED, MMD) and is comparable on
mode entropy and off-support. The off-support residual is edge-smoothing of the
continuous density, which EMA does not address.
