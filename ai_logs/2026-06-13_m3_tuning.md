Find slightly better M3 DMD hyperparameters before M4, without redesigning M3.

make the clean M3 pipeline depend on tuning infrastructure.

## Temporary tuning design
- Dependency direction is tuning -> M3 only: the script imports and reuses
  `src.train_dmd.train_variant` / `resolve_device`, `src.dmd`, `src.metrics`,
  `src.plotting`, `src.distributions`, `src.diffusion.load_teacher`,
  `src.utils`. Nothing in `src/` imports the tuning code.
- Not wired into `run.sh`; invoked as
  `python -m scripts.tune_m3_hparams --config configs/dmd_tuning.yaml`.
- Reuses the shared 2000-sample `outputs/eval_noise.pt` for plots and a separate
  fixed 10000-sample noise (`outputs/m3_tuning/eval_noise_10k.pt`) for metrics;
  the shared file is left untouched. Each trial reseeds to the same noise stream
  for a fair comparison.

## Search space (~15 curated trials, one factor at a time around `dmd_no_reg`)
- `lambda_reg` in {0.0, 0.025, 0.05, 0.10, 0.25}; `reg_loss_type` in {mse, smooth_l1, l1} (only if `lambda_reg>0`).
- `fake_updates_per_gen` (TTUR) in {1, 2, 3, 5}.
- `gen_lr` in {5e-5, 1e-4, 2e-4}; `fake_lr` in {5e-4, 1e-3, 2e-3}.
- one combo: `lambda_reg=0.05` (mse) + `fake_updates_per_gen=3`.
- `t_gen` fixed at `T-1`; DMD noise-time range fixed at the M3 baseline (0.02-0.98).

## Ranking metric
- Hard gate (mode coverage primary): reject `recovered_mode_count < 8` or `normalized_mode_entropy < 0.98` (penalized, still reported).
- Composite among valid trials: `score = energy_distance + mmd_rbf + 0.05 * off_support_fraction`.
- Justification: the brief's 0.25 off-support weight would dominate ED+MMD at M3
  magnitudes (0.045 vs ~0.008), inverting the intended priority; 0.05 keeps it a
  true tie-breaker. Both 0.05 and 0.25 composites plus raw metrics are reported.
